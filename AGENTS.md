# AGENTS.md

## Область действия

Эти правила действуют для всего репозитория RuSwitcher. Это Swift 6 / SwiftPM-приложение строки меню для macOS 13+ на AppKit, Carbon, CoreGraphics и ServiceManagement.

`RuSwitcher.app`, `.build/`, `*.dmg` и `*.zip` — генерируемые артефакты. Никогда не редактируйте содержимое `.app` напрямую: меняйте исходники и пересобирайте пакет.

## Ориентиры по коду

- `Sources/RuSwitcher/main.swift`, `AppDelegate.swift` — запуск, жизненный цикл, меню и мастер разрешений.
- `KeyboardMonitor.swift` — event tap, триггеры и краткоживущий буфер нажатий.
- `TextConverter.swift` — удаление, Unicode-вставка и обратная конвертация.
- `AutoSwitch.swift` — проверка безопасного контекста Accessibility/Secure Input.
- `DynamicKeyMapping.swift`, `KeyMapping.swift`, `KeyCodes.swift` — преобразование физических клавиш между раскладками.
- `LayoutSwitcher.swift`, `PerAppLayoutManager.swift` — раскладки и их состояние по приложениям.
- `SettingsManager.swift`, `SettingsWindowController.swift` — `UserDefaults` и UI настроек.
- `Localization.swift` — встроенные строки для 16 языков.
- `build_app.sh` — сборка, создание и подпись `.app`; `create_dmg.sh` — релизная упаковка и нотариализация.

## Обязательные инварианты

- Конвертация остаётся только ручной, после явного нажатия триггера. Не добавляйте словарную автокоррекцию.
- Не добавляйте сеть, телеметрию, обновлятор или внешние зависимости без явного требования.
- Никогда не записывайте набранный текст в логи или на диск и не используйте системный буфер обмена для конвертации. `rslog` намеренно является no-op.
- Сохраняйте очистку буфера нажатий после 15 секунд бездействия и текста обратной конвертации после 5 секунд.
- Перед каждым синтезированным событием сохраняйте проверки master toggle, Secure Event Input, защищённого AX-элемента, приложения и identity целевого поля. При смене контекста сбрасывайте чувствительное состояние.
- Помечайте синтезированные события `kRuSwitcherEventMarker`, чтобы монитор их игнорировал.
- Не ослабляйте блокировку менеджеров паролей и Apple Remote Desktop/Screen Sharing.
- UI и работа с AppKit должны соблюдать существующую изоляцию `@MainActor`; горячий путь event tap не должен выполнять тяжёлую работу.
- Новые пользовательские строки добавляйте через `L10n` для всех 16 поддерживаемых языков. Ключи `UserDefaults` сохраняйте совместимыми; миграцию старых ключей делайте явно.

## Стиль изменений

- Делайте минимальный целевой патч и следуйте существующему стилю Swift и русскоязычных комментариев.
- Не меняйте поведение триггеров, тайминги событий, модель разрешений или поддерживаемые macOS/архитектуры без необходимости задачи.
- Не редактируйте версию в `Info.plist` собранного пакета. `version.json` — единый источник версии, а `build_app.sh` штампует её в bundle.
- Не коммитьте секреты подписи/нотариализации. Конфигурация нотариализации находится вне репозитория.

## Проверка

Для обычной проверки компиляции используйте:

```bash
swift build
```

В репозитории пока нет test target. При изменении чистой логики добавляйте сфокусированные Swift-тесты, если это возможно без обращения к глобальному event tap или системным разрешениям.

Запускайте `bash build_app.sh` только когда нужно проверить реальный bundle: после изменения файлов он автоматически повышает patch-версию и build в `version.json`, удаляет прежний `RuSwitcher.app`, создаёт новый и подписывает его. После сборки проверьте архитектуры и подпись:

```bash
lipo -archs RuSwitcher.app/Contents/MacOS/RuSwitcher
codesign --verify --deep --strict --verbose=2 RuSwitcher.app
```

Изменения мониторинга клавиатуры, Accessibility, раскладок или вставки текста дополнительно проверяйте вручную с выданными Input Monitoring и Accessibility permissions: одиночный/двойной триггер, левые/правые модификаторы, комбинации, Caps Lock, конвертация после пробела, обратная конвертация, смена приложения/поля и защищённый ввод.

`create_dmg.sh` требует Developer ID, доступ к нотариализации и Finder UI. Не запускайте релизную упаковку без явного запроса на выпуск.
